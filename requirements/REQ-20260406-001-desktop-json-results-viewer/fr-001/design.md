# REQ-20260406-001:FR-001 — Load JSON Search Results

# Design Considerations
- The Desktop app MUST consume JSON generated by the existing Reporter structure (hash plus image path and sizeMB entries).
- Deserialization SHOULD target dedicated in-memory DTO/domain models instead of directly binding UI controls to raw JSON objects.
- The source JSON file MUST be treated as read-only input.

# Data Flow
1. User clicks "Open JSON Results" and chooses a JSON file through MAUI file picker.
2. Application service `IResultsLoader` validates and deserializes content to in-memory result models.
3. UI receives loaded model state and enables result navigation.

# Affected Components (Projects, Services, Classes)
- ImageDuplicationSearcher.Desktop: file picker UI and load command
- ImageDuplicateSearcher.Application: `IResultsLoader` and implementation
- Shared models: in-memory duplicate result model

# Dependencies
- System.Text.Json for deserialization
- MAUI FilePicker API for file selection

# Implementation Steps
1. Add in-memory models that map Reporter JSON output fields.
2. Implement `IResultsLoader` in Application with validation and exception mapping.
3. Wire Desktop UI load command to Application service and display success/error state.
